# Документация по папке `frontend`

Папка `frontend` содержит клиентскую часть проекта — веб-интерфейс для персонала и администраторов, обеспечивающий визуализацию, управление и мониторинг работы торгового бота и связанных сервисов.

---

## Структура папки `frontend`

- `src/` — исходный код фронтенда

  - `pages/` — страницы Next.js приложения
  - `core/` — основные компоненты интерфейса, контексты и утилиты
    - `components/` — React компоненты UI
    - `context/` — React контексты для состояния и авторизации
    - `styles/` — стили в SCSS и модули CSS
  - `public/` — статические ресурсы (иконки, изображения)
- `README.md` — инструкция по установке и запуску фронтенда

---

## Основные компоненты и особенности

### 1. Страницы (`pages/`)

- Содержит страницу администратора для управления биржами (например, Binance), реализованную с использованием React, MobX и Mantine:
  - Включение авторизации через контекст `AuthProvider`.
  - Включение навигационной боковой панели `Sidebar`.
  - Использование iframe для отображения внутреннего API или информации с сервера.

### 2. Компоненты (`core/components`)

- **Layouts-parts**

  - `view.tsx`Компонент, отвечающий за основной блок отображения текущего представления с хлебными крошками и условным отображением содержимого.
  - `page-view.tsx`Обертка страницы, которая контролирует отображение основной панели и сайдбара. Управляет языковой локализацией и роутингом.
  - `sidebar.tsx`
    Компонент бокового меню навигации.
- **Context**

  - `authProvider.tsx`
    Контекст и провайдер авторизации, управляющий состоянием пользователя и его правами.

### 3. Стили (`core/styles` и `styles/`)

- Используются SCSS-модули для изоляции стилей компонентов.
- Основные стили в `app.scss` включают оформление форм, кнопок, полей ввода и прочих UI-элементов с поддержкой вариантов отображения для разных страниц и режимов.

### 4. Локализация

- Используется библиотека `next-localization` для поддержки многоязычности и динамической смены языка интерфейса.

---

## Технологии

- React с функциональными компонентами и хуками (useState, useEffect).
- Менеджмент состояния – MobX.
- UI-библиотека Mantine для компонентов интерфейса.
- Next.js как SSR фреймворк с маршрутизацией по файлам.
- SCSS для кастомных стилей с модульной системой.
- Асинхронная работа с API через fetch или axios (предположительно, в отдельных сервисах не показано).
- Интернационализация через `next-localization`.

---

## Руководство пользователя и запуска

- В документации (`frontend/README.md`) описана стандартная установка:
  1. Установка Docker, Docker-compose и Make.
  2. Клонирование репозитория и копирование `.env` файлов.
  3. Запуск команд `make network init build up` и `make status`.
- Фронтенд запускается в рамках docker-контейнера, предоставляя персоналу веб-интерфейс.
- Интерфейс веб-панели позволяет контролировать состояния сервисов, просматривать логи и управлять ордерами.

---

## Рекомендации по разработке и расширению

- Создавайте новые страницы в `pages/` с использованием компонентов из `core/components`.
- Для управления состоянием расширяйте MobX хранилища и интегрируйте в компоненты через observer.
- Локализацию расширяйте добавлением новых ключей в файлы локалей.
- Для добавления новых элементов UI используйте Mantine компоненты и переопределяйте стили через SCSS-модули.
- Для взаимодействия с backend расширяйте существующие API вызовы или создавайте новые сервисы.
